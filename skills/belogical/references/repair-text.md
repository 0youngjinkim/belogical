# repair-text — Repairing existing text

> Role: extract the pyramid from already-written text, diagnose its defects, and correct the composition without changing the meaning.
> Loaded on: repair. For the group and summary criteria used in diagnosis, see grouping-order.md.
> Invariant: meaning, figures, and nuance (level of confidence, emphasis, implication) do not change. What makes text easier to understand is arrangement, not sentences — preserve sentences as far as possible.

## 1. Extract — pull out the text's actual structure

1. Find the reader's question this text is trying to answer.
2. Find whether an apex sentence (the core answer) exists, and where it sits.
3. Pull one conclusion sentence from each paragraph and draw the pyramid the text currently forms.

If the structure won't draw, that is itself the first diagnosis — there is no apex, or paragraphs lack conclusion sentences, or no groups have formed.

## 2. Diagnose — defect checklist

Inspect the extracted pyramid from four directions:

- **Composition**: no apex / the conclusion is buried at the end / information is listed in the order it was gathered or occurred
- **Summaries**: a higher-level sentence is an empty assertion ('there are N...') / the parent fails to summarize its children
- **Groups**: different kinds mixed in one group / overlaps or gaps (MECE violation) / a group so large its kinship has thinned / no logic to the order
- **Delivery**: headings are category labels / a message sent with a subject line has none, or one that names only the topic / questions raised before the text is ready to answer them, or answered before they were asked / the introduction argues or piles up new information

Use numeric thresholds (group size, etc.) as suspicion signals, not verdict lines — if a group looks big, confirm the kinship has actually thinned before ruling it a defect.

## 3. Rearrange — prescription by defect

| Defect | Prescription |
|---|---|
| Buried conclusion | Pull the apex sentence to the head; redeploy the rest as support |
| Empty assertion | Find what the children share and restate it substantively |
| Mixed kinds / MECE violation | Reform the groups — recheck that one plural noun covers each |
| Orderless sequence | Reorder by the grouping's basis (time, structure, degree) |
| Category-label headings / missing or topic-only subject line | Replace with headings that carry the message — for a subject line, the ask and any deadline |
| Defective introduction | Push argument and new information down into the body; rebuild it into a Situation–Complication that only reminds |

## 4. Check

- Does it pass 'Always check before you deliver' in SKILL.md?
- Compare before and after: are meaning, figures, and nuance preserved — no dropped information, no added claims, no shifted level of confidence? A primary option with a fallback stays primary — flattening 'send by Monday 10 a.m., or a draft Sunday night if that's tight' into two equal choices shifts the emphasis.
